**1. Update `format/2` in `birch_erlang_logger_ffi.erl`:**

Wrap the primary `format/2` clause body in a try/catch that returns a clean fallback on any exception. The fallback preserves birch-style formatting so the output is still useful:

```erlang
format(#{level := Level, msg := Msg, meta := Meta} = _Event, #{format_fn := FormatFn} = _Config) ->
    try
        LogRecord = case maps:get(birch_log_record, Meta, undefined) of
            undefined ->
                build_log_record_from_otp(Level, Msg, Meta);
            Record ->
                Record
        end,
        Formatted = FormatFn(LogRecord),
        [Formatted, $\n]
    catch
        Class:Reason:_Stacktrace ->
            %% Formatter crashed -- emit a fallback that preserves visibility.
            %% Includes FORMATTER_ERROR marker so the crash is obvious in logs.
            Timestamp = format_timestamp(maps:get(time, Meta, undefined)),
            FallbackMsg = try format_msg(Msg, Meta)
                          catch _:_ -> iolist_to_binary(io_lib:format("~p", [Msg]))
                          end,
            [Timestamp, <<" | ">>,
             level_to_string(Level), <<" | FORMATTER_ERROR | ">>,
             FallbackMsg, <<" | crash: ">>,
             iolist_to_binary(io_lib:format("~p:~p", [Class, Reason])),
             $\n]
    end;
```

Keep the second `format/2` clause (the fallback for missing format_fn) unchanged.

Key design decisions:
- The fallback format uses the same helper functions (format_timestamp, level_to_string, format_msg) which are independently tested and simpler than the full pipeline
- format_msg itself is wrapped in a nested try/catch in case it contributed to the crash
- The `FORMATTER_ERROR` marker makes crashes immediately visible in log output
- The crash class and reason are included for debugging
- No internal :logger logging (the `?LOG_INTERNAL` macro is private to OTP)

**1. Fix `format_msg/2` in `birch_erlang_logger_ffi.erl`:**

The 1-arg and 2-arg `report_cb` handlers have swapped return type handling. Fix per OTP spec:

Replace the `{report, Report}` clause of `format_msg/2`:

```erlang
format_msg({report, Report}, Meta) ->
    case maps:get(report_cb, Meta, undefined) of
        Fun when is_function(Fun, 1) ->
            %% 1-arg form: returns {io:format(), [term()]}
            try
                {Format, Args} = Fun(Report),
                unicode:characters_to_binary(io_lib:format(Format, Args))
            catch _:_ ->
                unicode:characters_to_binary(io_lib:format("~p", [Report]))
            end;
        Fun when is_function(Fun, 2) ->
            %% 2-arg form: returns unicode:chardata()
            try
                unicode:characters_to_binary(
                    Fun(Report, #{single_line => true, depth => 30, chars_limit => unlimited}))
            catch _:_ ->
                unicode:characters_to_binary(io_lib:format("~p", [Report]))
            end;
        _ ->
            unicode:characters_to_binary(io_lib:format("~p", [Report]))
    end;
```

Note the key changes:
- 1-arg: Now does `{Format, Args} = Fun(Report)` then `io_lib:format(Format, Args)` (was doing `unicode:characters_to_binary(Fun(Report))`)
- 2-arg: Now does `unicode:characters_to_binary(Fun(Report, Config))` (was doing `{Format, Args} = Fun(Report, Config)`)
- 2-arg config now includes `chars_limit => unlimited` per OTP convention
